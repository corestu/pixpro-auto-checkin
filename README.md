# PixPro 自动签到

> 针对 PixPro 站点的 WordPress 自动签到插件，支持主动任务与被动任务，签到失败自动邮件通知站长。

## 功能概览

- **定时签到**：通过 WP Cron 每日自动执行签到，可在后台自定义执行时间（时/分）。
- **主动任务**：签到成功后自动执行收藏、点赞片刻、评论点赞、关注/取关用户、发送私信等操作，获取额外经验。
- **被动任务**：使用第二个账号关注主账号、点赞主账号评论，让主账号获得被关注/被点赞经验。
- **失败邮件通知**：签到失败、主动/被动任务出现失败或异常时，自动发送邮件到站长邮箱。
- **IP 直连**：支持填写目标站点 IP，通过 `CURLOPT_RESOLVE` 绕过 DNS 直连目标服务器。
- **后台调试**：提供连接测试、签到测试、主动/被动任务测试、补挂区（单独执行某个操作）、测试邮件、清除记录等功能。
- **REST API**：提供 `/wp-json/ppo-checkin/v1/run` 接口供外部调用触发签到。
- **Cookie 有效期预估**：自动解析 Cookie 中的过期时间并在后台提示。

## 环境要求

| 项目 | 要求 |
|------|------|
| WordPress | 5.0+ |
| PHP | 7.2+ |
| PHP 扩展 | cURL |

## 安装

1. 将 `pixpro-auto-checkin.php` 上传到 `/wp-content/plugins/` 目录（或创建 `pixpro-auto-checkin` 文件夹放入）。
2. 或将 `pixpro-auto-checkin.php` 压缩为 zip 包，在 WordPress 后台「插件 → 安装插件 → 上传插件」选择 zip 文件安装。
3. 在 WordPress 后台「插件」页面激活 **PixPro 自动签到**。
4. 进入「设置 → PixPro 签到」进行配置。

## 配置说明

### 基础配置

| 字段 | 说明 |
|------|------|
| 登录 Cookie | 登录目标网站后，从请求标头中复制完整 Cookie 字符串贴入 |
| 目标站点地址 | 需要签到的网站首页地址，如 `https://pix.plus` |
| 网站 IP（选填） | 目标站点的 IP 地址，用于 IP 直连 |
| 执行时间 | 每天自动签到的时和分，秒数自动随机 |
| 失败邮件通知 | 勾选后，签到/任务失败时发送邮件到站长邮箱 |

### 主动任务配置

| 字段 | 说明 |
|------|------|
| 片刻 ID | 一键任务用此 ID 对片刻进行点赞/收藏操作 |
| 评论 ID | 一键任务用此 ID 对评论进行点赞/取消操作 |
| 关注用户 ID | 一键任务用此 ID 进行关注/取消关注操作 |
| 私信用户 ID | 主动任务用此 ID 发送私信（经验+2） |

### 被动任务配置

| 字段 | 说明 |
|------|------|
| 第二个账号 Cookie | 被动任务需要第二个账号来关注主账号、点赞主账号评论 |
| 被关注用户 ID | 第二个账号要关注的用户 ID（填写主账号的用户 ID） |
| 被点赞评论 ID | 第二个账号要点赞的评论 ID（填写主账号的评论 ID） |

## 后台功能

| 功能 | 说明 |
|------|------|
| 立即签到 | 一键执行全流程：签到 → 主动任务 → 被动任务 |
| 签到测试 | 仅检测连通性，不消耗每日签到 |
| 主动任务测试 | 单独执行主动任务并查看结果 |
| 被动任务测试 | 单独执行被动任务并查看结果 |
| 补挂区 | 单独执行某个操作（签到/收藏/点赞/评论点赞/关注/私信/被动关注/被动评论点赞），可指定次数 |
| 测试邮件 | 验证邮件通知功能是否正常 |
| 清除记录 | 清除上次执行记录 |
| 签到记录 | 显示上次执行的时间、签到/主动/被动任务结果 |

## REST API

```
GET /wp-json/ppo-checkin/v1/run
```

需要管理员登录态，触发一次完整签到流程。

## 主流程与邮件通知

```
WP Cron 定时触发
    │
    ▼
签到请求
    ├── 失败 ──→ 邮件通知（签到失败原因）
    │
    └── 成功
        ├── 主动任务（收藏/点赞/评论/关注/私信）
        │   ├── 部分失败 ──→ 邮件通知（失败详情）
        │   └── 异常 ──→ 邮件通知（异常信息）
        │
        └── 被动任务（被关注/被点赞）
            ├── 部分失败 ──→ 邮件通知（失败详情）
            └── 异常 ──→ 邮件通知（异常信息）
```

流程中任何一处失败或异常都会触发邮件通知，主动任务与被动任务相互独立，一方异常不影响另一方执行。
